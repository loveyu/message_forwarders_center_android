# FlowGate

**English** | [中文](README_CN.md)

Android Message Forwarding Center - Always-on Service Architecture

## Features

### Core Architecture
- **Android Foreground Service** - START_STICKY ensures service survival
- **Unified Ticker Scheduler** - Single scheduler drives all periodic checks with event-driven triggers
- **Real-time Network Monitoring** - ConnectivityManager.NetworkCallback
- **Event-driven** - Network changes, screen on, user unlock, charging state, app foreground/background

### Link Layer (LinkManager)
| Type | Library | Features |
|------|---------|----------|
| MQTT | Eclipse Paho | TLS, consecutive failure backoff |
| WebSocket | OkHttp | WSS/TLS, Basic Auth, Gotify protocol, message replay |
| TCP | Kotlin Socket | Keep-Alive, frame protocols (lb/tlv/split) |

Link type is auto-detected via DSN protocol (`mqtt://`, `ws://`, `tcp://`). Connection parameters are passed through URL query strings. Supports TLS certificate configuration (`file://`, `sdcard://`, `data://`, `http(s)://`).

### Input Layer (InputManager)
| Type | Description |
|------|-------------|
| HTTP | NanoHTTPD server with Basic/Bearer/Query authentication |
| Link Subscription | Subscribe to MQTT/WebSocket/TCP link messages |
| Gotify Replay | Replay offline messages via REST API after reconnection |

### Queue Layer (QueueManager)
| Type | Features |
|------|----------|
| Memory Queue | High-performance, configurable capacity, overflow strategies (dropOldest/dropNew/block) |
| SQLite Queue | Persistent storage, retry with exponential backoff, auto cleanup |

### Output Layer (OutputManager)
- **HTTP**: POST/PUT/GET with configurable retry
- **Link Publish**: MQTT/WebSocket/TCP publishing, multi-linkId fan-out
- **Internal**: Clipboard, File, Broadcast, Notify

Each output supports instant retry (`retry`) and async failure queue (`onFailureQueue`).

### Rule Engine (RuleEngine)
Execution order: `call -> decode -> detect -> enrich -> filter -> extract -> format -> output`

| Option | Description |
|--------|-------------|
| `call` | Invoke named HTTP call resources, inject results as variables for subsequent steps |
| `decode` | Pipeline decoding (`base64Decode|jsonDecode`) |
| `detect` | Type detection (`image`/`json`/`text`) |
| `enrich` | Data enrichment (Gotify icon, etc.) |
| `filter` | Expression filtering |
| `extract` | GJSON path extraction |
| `format` | Template formatting |

Supports `onError` error-handling pipelines and `when`/`deny` network condition control.

### Dead Letter Queue (DeadLetter)
Messages that exhaust all retries enter the dead letter queue with customizable pipeline for final processing. Dead letter messages are saved to `data://deadletter/` and auto-restored after restart.

### Scheduler
Unified Ticker drives all periodic checks (default 40s, shorter when charging). Event triggers are protected by a 5-second minimum debounce.

### Network Condition Control
`when`/`deny` support `network` (network type), `ssid` (WiFi name, regex supported), `bssid` (MAC address), `ipRanges` (CIDR) conditions. Applicable to link, input, output, and rule configurations.

## Status Bar / Service Status Display

The notification bar (refreshed by the unified Ticker, default 40s) displays runtime statistics and brief status indicators.

Display format:
- L: Link count
- I: Input count
- O: Output count

Example: `L3 I2 O4`

Status flags (separated by `|`, shown as needed):
- `Pause Receive` - Receiving paused
- `Pause Forward` - Forwarding paused
- `WLock` - WakeLock active
- `WiFiLock` - WiFi Lock active

Example: `L3 I2 O4 | Pause Forward | WLock`

## Permissions

- `INTERNET` - Network communication
- `ACCESS_NETWORK_STATE` - Network state monitoring
- `ACCESS_WIFI_STATE` - WiFi state monitoring
- `NEARBY_WIFI_DEVICES` - WiFi info access (Android 13+ for SSID/BSSID)
- `ACCESS_FINE_LOCATION` - Location permission (required for WiFi info on some devices)
- `ACCESS_BACKGROUND_LOCATION` - Background location (Android 10+ for background WiFi SSID/BSSID)
- `FOREGROUND_SERVICE` - Foreground service
- `FOREGROUND_SERVICE_REMOTE_MESSAGING` - Remote messaging foreground service type (Android 14+, for MQTT/WebSocket/TCP long connections)
- `WAKE_LOCK` - Wake lock (keep CPU running with screen off)
- `RECEIVE_BOOT_COMPLETED` - Boot auto-start
- `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` - Battery optimization exemption
- `POST_NOTIFICATIONS` - Notification permission (Android 13+)
- `READ_MEDIA_IMAGES` - Read media images (Android 13+)
- `READ_EXTERNAL_STORAGE` - Read external storage (Android 12 and below)
- `WRITE_EXTERNAL_STORAGE` - Write external storage (Android 10 and below)
- `MANAGE_EXTERNAL_STORAGE` - Manage all files (Android 11+, for sdcard:// path writes)
